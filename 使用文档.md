# OpenClaw 一键部署工具 - 使用文档

## 一、软件介绍

**OpenClaw 一键部署工具** 是一款面向小白的桌面应用，用于零门槛部署 OpenClaw 私人 AI 助手。基于 Tauri 2 + React + TypeScript 构建，提供图形化配置界面，无需手写配置文件。

### 主要功能

- **环境检测**：自动检测 Node.js（≥22）、npm、OpenClaw 安装状态
- **一键安装**：通过 npm 全局安装 OpenClaw
- **配置向导**：支持 Claude、GPT、DeepSeek、阿里云百炼、Kimi 等，可配置自定义 API 地址（如 OneAPI、NewAPI 等中转服务）
- **启动服务**：一键启动/停止 OpenClaw Gateway
- **一键打开对话界面**：启动后可直接打开 Web 对话界面
- **渠道配置**：支持 Telegram、飞书、QQ 等渠道的图形化配置与配对

### 推荐使用流程

1. 安装 Node.js（建议 22 及以上）
2. 打开本工具，按向导完成「安装 OpenClaw」→「配置 AI 模型」→「启动服务」
3. 点击「一键打开 OpenClaw 对话界面」进入对话
4. 如需 Telegram 等渠道，在「配置渠道」中完成配对

## 二、Shell 安装脚本介绍

除本桌面工具外，还提供 **OpenClaw_Shell_Install.cmd** + **OpenClaw_Shell.ps1** 脚本，用于在命令行中安装并启动 OpenClaw。**脚本不会退出**，提供交互式菜单循环。

### 脚本文件

- `OpenClaw_Shell_Install.cmd`：入口，自动调用 PowerShell 菜单
- `OpenClaw_Shell.ps1`：交互式菜单（需与 CMD 同目录）

### 主菜单功能

| 选项 | 功能 | 说明 |
|------|------|------|
| 1 | 快速配置 | 1a=交互式 onboard；1b=快速配置（选提供商→自动拉取模型列表→选模型+API，与软件一致） |
| 2 | 启动 Gateway | 启动 OpenClaw 网关服务 |
| 3 | 常用命令 | 子菜单：gateway status/start/stop、models list、channels status、doctor 等 |
| 4 | 运行 OpenClaw | 启动 OpenClaw 主程序 |
| 5 | 更新 OpenClaw | 自动检测实际使用的安装路径，执行 `npm install -g openclaw@latest --prefix <安装目录>` 升级 |
| 6 | 打开对话界面 | 启动 Gateway 并打开浏览器访问对话界面 |
| 7 | 查看配置路径 | 显示 `~/.openclaw` 及配置文件列表 |
| 8 | 帮助文档 | 显示 docs.openclaw.ai 链接并打开浏览器 |
| 0 | 退出 | 退出脚本 |

### 常用命令子菜单（中文说明）

- **gateway status**：查看 Gateway 运行状态
- **gateway start/stop/restart**：启动 / 停止 / 重启 Gateway
- **models list**：列出已配置的 AI 模型
- **channels status**：查看渠道（Telegram、飞书等）状态
- **doctor**：环境检测与修复
- **status**：查看整体状态

### 使用方式

- 双击 `OpenClaw_Shell_Install.cmd` 或桌面快捷方式 `OpenClaw Shell 安装.lnk`
- 或在 CMD 中执行脚本路径

### 前置条件

- 已安装 Node.js（含 npm）
- 网络可访问 npm registry

## 三、常见问题

### 1. 一键打开对话界面打开的是文档文件夹（或错误页面）

**现象**：点击「一键打开 OpenClaw 对话界面」后，浏览器打开的是文档目录或其他页面，而不是对话界面。

**原因**：系统默认浏览器或外部打开逻辑可能将 URL 解析错误，或打开了错误的目标。

**解决**：

- 手动在浏览器中访问：`http://127.0.0.1:18789/?token=<你的token>`
- Token 会在点击时复制到剪贴板，可在 Control UI 设置中粘贴
- 对话界面正确地址示例：`http://127.0.0.1:18789/__openclaw__/canvas/`

---

### 2. 打开经典终端配置界面失败

**现象**：提示 `"C:\Users\Administrator\AppData\Roaming\npm\openclaw.cmd" 不是内部或外部命令，也不是可运行的程序或批处理文件`。

**原因**：

- OpenClaw 未通过 `npm install -g openclaw` 成功安装
- `%APPDATA%\npm` 未加入系统 PATH
- 安装目录权限异常

**解决**：

- 使用桌面上的 **OpenClaw Shell 安装** 快捷方式，先完成全局安装
- 或在 CMD 中执行：`npm install -g openclaw`
- 确认 `%APPDATA%\npm` 在 PATH 中，且该目录下存在 `openclaw.cmd`

---

### 3. 内存资源不足，无法处理此命令

**现象**：执行 openclaw 相关命令时，提示「内存资源不足，无法处理此命令」。

**原因**：系统可用内存不足，或进程/句柄过多。

**解决**：

- 关闭不必要的程序，释放内存
- 重启电脑后再试
- 检查是否有其他占用大量内存的进程

---

### 4. 404 错误

**现象**：访问对话界面或 Control UI 时出现 404 页面。

**原因**：

- Gateway 未成功启动
- 访问的 URL 路径错误
- 服务刚启动，尚未就绪

**解决**：

- 确认已点击「启动 Gateway」，并等待几秒
- 正确地址：`http://127.0.0.1:18789/`（需带 token 参数）
- 若仍 404，检查 Gateway 是否在运行，可查看终端日志中的 `[gateway] listening on` 等信息

---

### 5. OpenClaw Gateway 终端窗口一直出现

**现象**：启动后会出现一个显示日志的黑色终端窗口。

**说明**：这是 Gateway 的日志输出窗口，用于查看运行状态和排错。若希望减少干扰，可最小化该窗口，或后续通过配置改为后台服务方式运行。

---

### 6. WebChat 连接后很快断开（code=1001）

**现象**：日志中出现 `[ws] webchat disconnected code=1001`。

**原因**：客户端主动关闭或网络不稳定。

**解决**：刷新对话页面重新连接；若频繁断开，检查网络和防火墙设置。

---

### 7. 未检测到 npm

**现象**：Shell 脚本提示「未检测到 npm」。

**解决**：安装 Node.js（建议从官网下载 LTS 版本），安装时勾选「Add to PATH」。安装完成后重启终端再试。

---

### 8. 脚本更新显示成功，但对话界面仍显示旧版本

**现象**：Shell 脚本选择「更新 OpenClaw」后提示成功，但打开对话界面或执行 `openclaw status` 仍显示旧版本。

**原因**：若通过**一键部署**安装到自定义目录（如 `C:\Users\...\openclaw`），而脚本之前固定更新 `%APPDATA%\npm`，会导致更新到错误位置，对话界面使用的仍是自定义目录下的旧版本。

**解决**：新版脚本已修复，会优先检测系统 PATH 中实际调用的 openclaw 路径，并更新到该安装目录。请使用最新版脚本重新更新。若仍有问题，可在一键部署工具中查看「安装路径」，手动执行：`npm install -g openclaw@latest --prefix <你的安装路径>`。

## 四、技术支持

- 项目路径：`C:\Users\Administrator\Desktop\openclow`
- 开发模式：`npm run tauri dev`
- 构建发布版：`npm run tauri build`

构建完成后，可执行文件位于：`src-tauri/target/release/openclaw-deploy.exe`
