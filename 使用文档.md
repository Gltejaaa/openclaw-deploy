## 二、桌面工具功能说明

### 主要功能

| 功能 | 说明 |
|------|------|
| 环境检测 | 检测 Node.js、npm、Git、OpenClaw；PATH 未配置时提供一键添加 |
| 一键安装 | 通过 npm 全局安装 OpenClaw，自动写入 PATH |
| AI 配置 | 支持 Claude、GPT、DeepSeek、Kimi、通义千问、阿里云百炼；可自定义 API 地址（OneAPI、NewAPI 等） |
| 启动服务 | **启动 Gateway 并自动打开对话网页**：启动后浏览器自动打开对话界面 |
| 前台启动 | 计划任务失败时，可用「前台启动 Gateway」在新窗口运行 |

### 推荐使用流程

1. 安装 Node.js（若未安装）
2. 打开本工具，依次完成：**环境检测** → **安装 OpenClaw** → **配置 AI 模型**
3. 点击 **「启动 Gateway 并自动打开对话网页」**，等待浏览器自动打开
4. 如需 Telegram 等渠道，在「启动服务」页的渠道卡片中完成配置与配对

## 三、Shell 脚本（命令行方式）

除桌面工具外，还提供命令行脚本，支持 **Windows** 与 **Linux/macOS**。

### Windows 脚本

| 文件 | 说明 |
|------|------|
| `OpenClaw_Shell_Install.cmd` | 入口，双击运行后调用 PowerShell 菜单 |
| `OpenClaw_Shell.ps1` | 交互式菜单（需与 CMD 同目录） |
| `OpenClaw_Shell_Bootstrap.ps1` | UTF-8 启动器 |

> ⚠️ **完整功能**：需复制整个 `scripts` 文件夹（含 `.ps1` 文件），否则仅有最小安装功能。

### Linux / macOS 脚本

| 文件 | 说明 |
|------|------|
| `install.sh` | 一键安装；`curl ... \| bash -s menu` 进入交互式菜单 |
| `OpenClaw_Shell.sh` | 交互式菜单（与 Windows 功能对应） |

**使用方式**：

```bash
# 一键安装（线性流程）
curl -fsSL https://raw.githubusercontent.com/3445286649/openclaw-deploy/main/install.sh | bash

# 交互式菜单（需 clone 或下载 release 后）
./scripts/OpenClaw_Shell.sh

# 或通过 install 传入 menu 参数
curl -fsSL .../install.sh | bash -s menu
```

### 主菜单

| 选项 | 功能 |
|:---:|------|
| 1 | 快速配置（交互式 onboard / 选提供商+模型+API） |
| 2 | 启动 Gateway |
| 3 | 常用命令（status、start、stop、doctor 等） |
| 4 | 运行 OpenClaw 主程序 |
| 5 | 更新 OpenClaw |
| 6 | 打开对话界面（启动 Gateway 并打开浏览器） |
| 7 | 配置路径（设置自定义路径、添加 PATH、安装在其他盘指引） |
| 8 | 帮助与文档 |
| 10 | Gateway 相关（修复任务、前台启动等） |

### 常用命令

```bash
openclaw gateway start    # 启动 Gateway
openclaw gateway stop     # 停止 Gateway
openclaw gateway status   # 查看状态
openclaw doctor           # 环境检测
openclaw onboard          # 交互式配置
```

## 四、常见问题

### 1. 启动后浏览器打开的是错误页面（文档夹、空白页等）

**现象**：点击「启动 Gateway 并自动打开对话网页」后，浏览器未正确打开对话界面。

**解决**：手动在浏览器地址栏输入：`http://127.0.0.1:18789`，按回车访问。

---

### 2. openclaw 不是内部或外部命令

**现象**：CMD 中执行 `openclaw` 提示找不到命令。

**原因**：`%APPDATA%\npm` 未加入系统 PATH，或 OpenClaw 未正确安装。

**解决**：
- 在桌面工具的「环境检测」页点击「一键添加 PATH」
- 或使用 Shell 脚本菜单 [7] 配置路径
- 添加后需**新开 CMD 窗口**才生效

---

### 3. Gateway 启动失败 / RPC probe failed

**现象**：`openclaw gateway start` 后状态仍显示异常。

**原因**：`gateway.mode` 未设置（OpenClaw 2026+ 要求）。

**解决**：在 CMD 中执行 `openclaw config set gateway.mode local`，或重新运行桌面工具的安装/配置流程。

---

### 4. 热迁移后 Gateway 无法启动

**现象**：将 OpenClaw 复制到新目录并删除源目录后，Gateway 打不开。

**原因**：计划任务中的 `gateway.cmd` 仍指向已删除的旧路径。

**解决**：执行 `openclaw gateway install --force` 强制重新生成，或使用桌面工具的「前台启动 Gateway」作为临时方案。

---

### 5. 404 错误

**现象**：访问 `http://127.0.0.1:18789` 出现 404。

**原因**：Gateway 未成功启动，或服务尚未就绪。

**解决**：
- 确认已点击「启动 Gateway 并自动打开对话网页」
- 等待几秒后再访问
- 检查是否有防火墙/杀毒软件拦截

---

### 6. 内存资源不足

**现象**：执行 openclaw 命令时提示「内存资源不足，无法处理此命令」。

**解决**：关闭不必要的程序，重启电脑后再试。

---

### 7. 未检测到 npm

**现象**：Shell 脚本或桌面工具提示「未检测到 npm」。

**解决**：安装 Node.js（建议 LTS 版本），安装时勾选「Add to PATH」，安装完成后重启终端。

---

### 7.1 Linux 安装 OpenClaw 失败（npm 权限或网络）

**现象**：Shell 脚本提示「安装失败，请检查 npm 权限或网络」。

**解决**：
1. **权限**：尝试 `sudo npm install -g openclaw`，或脚本会自动切换用户目录安装（`~/.local/bin`）
2. **网络**：国内可尝试 `npm config set registry https://registry.npmmirror.com`
3. **手动安装**：`npm install -g openclaw` 后，确保 `~/.local/bin` 或 `$(npm config get prefix)/bin` 在 PATH 中

---

### 8. 更新后仍显示旧版本

**现象**：执行更新后，对话界面或 `openclaw status` 仍显示旧版本。

**原因**：若通过一键部署安装到自定义目录，更新可能写入了错误位置。

**解决**：在桌面工具中查看「安装路径」，手动执行：
```bash
npm install -g openclaw@latest --prefix <你的安装路径>
```

---

### 9. WebChat 连接断开（code=1001）

**现象**：日志中出现 `[ws] webchat disconnected code=1001`。

**原因**：客户端主动关闭或网络不稳定。

**解决**：刷新对话页面重新连接；若频繁断开，检查网络和防火墙。

---

### 10. Telegram 机器人无响应 / 不能对话

**现象**：Telegram 机器人收不到回复，或对话界面无响应。

**原因**：Gateway 未正常运行（RPC probe 失败），或配置路径不一致导致 Gateway 与部署工具使用不同目录。

**解决**：
1. **清空「自定义配置路径」**：留空使用默认 `~/.openclaw`，避免多目录冲突
2. **重新启动 Gateway**：点击「启动 Gateway 并自动打开对话网页」，或 CMD 执行 `openclaw gateway stop` 后 `openclaw gateway start`
3. **计划任务失败时**：使用「前台启动 Gateway」，在新窗口运行 `openclaw gateway` 并保持不关闭
4. **检查状态**：CMD 执行 `openclaw gateway status`，若显示 RPC probe failed，按上述步骤重试

---

### 11. OpenClaw 安装在其他盘，如何填写配置路径？

> Shell 脚本用户：菜单 [7] 配置路径 -> [a] 设置自定义配置路径，或 [d] 查看指引。

**现象**：OpenClaw 或配置目录在 D 盘、E 盘等非系统盘，不知道「自定义配置路径」该填什么。

**说明**：这里填的是**配置目录**（存放 openclaw.json、凭证、会话的文件夹），不是 OpenClaw 程序的安装目录。

**填写方法**：

1. **先点「自动检测」**：若检测到有效路径会自动填入
2. **手动查找**：
   - 打开资源管理器，找到包含 `openclaw.json` 的文件夹
   - 常见位置：`D:\openclaw\.openclaw`、`E:\openclaw-config`、`D:\Users\你的用户名\.openclaw`
   - 在地址栏复制完整路径，粘贴到输入框
3. **格式要求**：
   - 使用 `\` 或 `/` 均可，如 `D:\openclaw\.openclaw` 或 `D:/openclaw/.openclaw`
   - 末尾不要加反斜杠
   - 路径中必须存在 `openclaw.json` 文件

**示例**：

| 你的配置所在位置 | 应填写的路径 |
|------------------|--------------|
| `D:\openclaw\.openclaw` | `D:\openclaw\.openclaw` 或 `D:/openclaw/.openclaw` |
| `E:\my-openclaw-config` | `E:\my-openclaw-config` |
| `D:\Users\Admin\.openclaw` | `D:\Users\Admin\.openclaw` |
